# PRAMA CoreConfig Sensitivity Analysis

Generated by `scripts/prama_sensitivity_sweep.py`. Implementation lives in
`src/aptadynamik/observer/sensitivity.py`.

## Target Functional

`separation(cfg) = mean_integrity(A_varied, t in [60,80]) - mean_integrity(B_monotonic, t in [60,80])`,
computed by stepping both deterministic scenarios from `verification/scenarios.py` under the
given `cfg` (preserving the scenarios' `regime_geometry.lambda_recovery=0.10`,
`lambda_gain=0.03` setup). Positive separation means the varied/clean trajectory stays
viable while the monotonic-stress trajectory loses integrity.
