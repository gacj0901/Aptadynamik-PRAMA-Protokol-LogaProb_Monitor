# v0.2 — PRAMA Components: Dynamic Coupling Viability

This release refactors PRAMA Components from a failure-mode classifier into a trajectory viability monitor.

## Canonical fields

- `acople`
- `viability`
- `distance_to_threshold`
- `threshold_crossed`
- `viability_status`
- `boundary_pressure`
- `boundary_side`

## Removed canonical fields

- `rima`
- `mode`
- `collapsed`

## Interpretation

PRAMA Components measures dynamic coupling: the capacity of a generative trajectory to preserve continuity under admissible variation. Its primary output is distance to the inviability threshold, not forensic classification.

## Methodological constraint

The current implementation is substrate-blind. It measures logprob-derived generative geometry. It does not measure semantic truth, intention, material energy, GPU cost, temperature, latency, or memory unless exogenous telemetry is integrated.
